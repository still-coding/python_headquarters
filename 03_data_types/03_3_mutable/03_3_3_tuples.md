# 📦 Кортежи `tuple`
```toc
minLevel: 2
```
Кортеж - это упорядоченная **неизменяемая** коллекция объектов.

> ⚠️ Несмотря на то, что кортежи неизменяемы, мы разместили их здесь, так как они являются составным типом данных и часто используются вместе с изменяемыми типами.

## Создание кортежей

```python
# Пустой кортеж
empty = ()
also_empty = tuple()

# Кортеж с элементами
point = (3, 4)
colors = ("красный", "зеленый", "синий")

# Кортеж из одного элемента (обязательна запятая!)
single = (42,)
not_tuple = (42)    # Это число, а не кортеж!

# Кортеж без скобок (но лучше со скобками для читаемости)
coords = 10, 20
```

## Операции с кортежами

```python
point = (3, 4, 5)

# Доступ (как у списков)
point[0]           # 3
point[-1]          # 5

# Срезы
point[0:2]         # (3, 4)

# Длина
len(point)         # 3

# Проверка принадлежности
3 in point         # True

# Конкатенация
(1, 2) + (3, 4)    # (1, 2, 3, 4)

# Повторение
(1, 2) * 3         # (1, 2, 1, 2, 1, 2)

# Нельзя изменять!
# point[0] = 10    # TypeError!
```

## Распаковка кортежей

```python
point = (3, 4)

# Распаковка
x, y = point       # x = 3, y = 4

# Обмен значений
a = 10
b = 20
a, b = b, a        # a = 20, b = 10

# Множественное присваивание
name, age, city = ("Иван", 25, "Москва")

# Распаковка с *
first, *middle, last = (1, 2, 3, 4, 5)
print(first)   # 1
print(middle)  # [2, 3, 4]
print(last)    # 5
```

## Методы кортежей

Кортежи имеют всего два метода:

```python
numbers = (1, 2, 3, 2, 4, 2, 5)

# count() - подсчет вхождений
numbers.count(2)    # 3

# index() - поиск индекса первого вхождения
numbers.index(3)    # 2
numbers.index(2, 2) # 3 (поиск начиная с индекса 2)
```

## Когда использовать кортежи?

✅ **Используйте кортежи для:**
- Неизменяемых данных (координаты, RGB цвета)
- Ключей в словарях (списки нельзя!)
- Возвращения нескольких значений из функции
- Когда нужна гарантия неизменности
- Когда важна производительность (кортежи быстрее списков)

❌ **Не используйте кортежи когда:**
- Нужно изменять элементы
- Требуется добавлять/удалять элементы
- Работаете с данными, которые часто меняются

## Кортежи vs Списки

| Характеристика | Кортеж | Список |
|----------------|---------|---------|
| Изменяемость | ❌ Неизменяемый | ✅ Изменяемый |
| Синтаксис | `(1, 2, 3)` | `[1, 2, 3]` |
| Производительность | Быстрее | Медленнее |
| Методы | 2 метода | Много методов |
| Использование как ключ словаря | ✅ Можно | ❌ Нельзя |
| Размер в памяти | Меньше | Больше |

## Практические примеры

### Возврат нескольких значений из функции

```python
def get_user_info():
    name = "Иван"
    age = 25
    city = "Москва"
    return name, age, city  # Возвращается кортеж

# Распаковка результата
user_name, user_age, user_city = get_user_info()
print(f"{user_name}, {user_age} лет, {user_city}")
```

### Использование как ключей словаря

```python
# Координаты точек на плоскости
locations = {
    (0, 0): "начало координат",
    (1, 0): "точка на оси X",
    (0, 1): "точка на оси Y",
    (1, 1): "диагональная точка"
}

# Поиск локации
point = (1, 1)
print(locations[point])  # "диагональная точка"

# Списки использовать нельзя!
# locations[[1, 1]] = "test"  # TypeError: unhashable type: 'list'
```

### Обмен значений переменных

```python
# Старый способ (с временной переменной)
temp = a
a = b
b = temp

# Питоновский способ (с кортежами)
a, b = b, a  # Элегантно и читаемо!
```

### Неизменяемые конфигурации

```python
# Настройки подключения к БД (не должны меняться)
DB_CONFIG = (
    "localhost",  # host
    5432,         # port
    "mydb",       # database
    "user",       # username
)

host, port, database, username = DB_CONFIG
```

## Именованные кортежи (namedtuple)

Для более читаемого кода можно использовать `namedtuple`:

```python
from collections import namedtuple

# Создание именованного кортежа
Point = namedtuple('Point', ['x', 'y'])

# Использование
p = Point(3, 4)
print(p.x)  # 3
print(p.y)  # 4

# Можно использовать как обычный кортеж
print(p[0])  # 3

# Распаковка работает
x, y = p
```
