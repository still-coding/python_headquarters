# 📦 Имена и связывание
```toc

```

## В Python нет «переменных» в классическом смысле

Во многих языках программирования переменная — это **ячейка памяти**, в которую кладут значение. Как коробка с надписью: положил число в коробку `x`, достал из коробки `x`.

**В Python всё иначе.** Здесь правильнее говорить не «переменная», а **имя** (name). Имя — это **бирка**, которую мы вешаем на объект. Сам объект живёт где-то в памяти, а имя просто **ссылается** на него.

```python
>>> x = 42
```

Что здесь происходит:
1. Python создаёт **объект** `42` (целое число) в памяти
2. Привязывает к нему **имя** `x`

Это называется **связывание** (name binding) — оператор `=` связывает имя с объектом.

```python
>>> y = x
```

Теперь и `x`, и `y` — это **две бирки на одном и том же объекте**:

```python
>>> id(x)  # id() показывает уникальный адрес объекта в памяти
140234866221360
>>> id(y)
140234866221360  # Тот же адрес! Один объект, два имени.
```

## Тип принадлежит объекту, а не имени

В Python нет необходимости объявлять тип заранее. **Тип — это свойство объекта**, а не имени. Одно и то же имя можно привязать сначала к числу, потом к строке:

```python
>>> x = 42
>>> type(x)   # Спрашиваем: «На какой тип объекта ссылается x?»
<class 'int'>

>>> x = "hello"
>>> type(x)   # Теперь x ссылается на строку
<class 'str'>

>>> x = [1, 2, 3]
>>> type(x)   # А теперь на список
<class 'list'>
```

Это называется **динамическая типизация** — имя не привязано к типу, оно может указывать на объект любого типа.

> 💡 Слово «переменная» в Python всё-таки используют — для краткости. Но полезно помнить, что за ним стоит именно модель «имя → объект», а не «ячейка памяти».

## Связывание и пересвязывание

Когда мы пишем `=`, мы **привязываем** (или **пересвязываем**) имя:

```python
>>> a = 10      # Привязали имя a к объекту 10
>>> a = 20      # Пересвязали имя a — теперь оно указывает на объект 20
                # Объект 10 никуда не делся (пока), но имя a на него больше не указывает
```

### Что происходит с «брошенными» объектами?

Если на объект больше не указывает ни одно имя, Python автоматически удалит его из памяти. Это называется **сборка мусора** (garbage collection):

```python
>>> x = "hello"   # Создали объект "hello", привязали x
>>> x = "world"   # x пересвязали на "world"
                   # Если на "hello" больше никто не ссылается — Python его удалит
```

## Почему это важно: изменяемые объекты

Модель «имя → объект» особенно важно понимать при работе с **изменяемыми** типами данных ([[03_3_0_lists|списки]], [[03_3_3_dicts|словари]], [[03_3_5_sets|множества]]):

```python
>>> a = [1, 2, 3]
>>> b = a          # b — ещё одна бирка на ТОМ ЖЕ списке

>>> b.append(4)    # Изменяем список через имя b
>>> a              # Смотрим через имя a — список изменился!
[1, 2, 3, 4]

>>> id(a) == id(b)  # Это один и тот же объект
True
```

Это **не баг**, это следствие модели: `a` и `b` — два имени для одного объекта. Мы изменили объект — оба имени видят изменения.

Если нужна **независимая копия**:

```python
>>> a = [1, 2, 3]
>>> b = a.copy()    # Создали новый список с теми же элементами
>>> b.append(4)
>>> a               # a не изменился — это другой объект
[1, 2, 3]
>>> id(a) == id(b)
False
```

С **неизменяемыми** типами ([[03_2_0_numbers|числа]], [[03_2_3_strings|строки]], [[03_3_2_tuples|кортежи]]) такой проблемы нет — их нельзя изменить «на месте», можно только создать новый объект:

```python
>>> a = "hello"
>>> b = a
>>> b = b.upper()  # upper() создаёт НОВЫЙ объект "HELLO"
>>> a              # a по-прежнему ссылается на "hello"
'hello'
>>> b
'HELLO'
```

## Правила именования

### ✅ Допустимые имена

```python
name = "Иван"           # латинские буквы
user_name = "Ivan"       # snake_case (рекомендуемый стиль в Python)
age2 = 30                # цифры допустимы (но не в начале)
_private = "секрет"      # подчёркивание в начале
MAX_SIZE = 100           # UPPER_CASE для констант
```

### ❌ Недопустимые имена

```python
# 2name = "Иван"    # SyntaxError: нельзя начинать с цифры
# my-name = "Иван"  # SyntaxError: дефис — это оператор вычитания
# class = "Python"   # SyntaxError: class — зарезервированное слово
```

### 💡 Соглашения (PEP 8)

```python
# Переменные и функции — snake_case
user_name = "Иван"

# Константы — UPPER_SNAKE_CASE
MAX_RETRIES = 3

# Классы — CamelCase (об этом в разделе ООП)
# class UserProfile:
#     pass
```

## Множественное связывание

```python
# Одно значение — несколько имён
a = b = c = 0       # Все три имени указывают на один объект 0
print(a, b, c)       # 0 0 0

# Несколько связываний одной строкой
x, y, z = 1, 2, 3   # x→1, y→2, z→3
print(x, y, z)       # 1 2 3

# Обмен (swap) — идиоматичный Python
a = 10
b = 20
a, b = b, a          # Работает без третьей переменной!
print(a, b)          # 20 10
```

> 💡 В других языках для обмена значений обычно нужна временная переменная. Python вычисляет правую часть целиком, а потом связывает имена.

## Распаковка (Unpacking)

Распаковка — это связывание имён с элементами последовательности (списка, кортежа, строки):

### Базовая распаковка

```python
# Распаковка списка
coordinates = [55.75, 37.62]
latitude, longitude = coordinates
print(latitude)   # 55.75
print(longitude)  # 37.62

# Распаковка кортежа
person = ("Иван", 25, "Москва")
name, age, city = person
print(name)  # Иван

# Распаковка строки
first, second, third = "ABC"
print(first)  # A
```

> ⚠️ **Количество имён должно совпадать с количеством элементов:**
> ```python
> a, b = [1, 2, 3]  # ValueError: too many values to unpack
> a, b, c = [1, 2]  # ValueError: not enough values to unpack
> ```

### Распаковка со звёздочкой `*`

Звёздочка собирает «лишние» элементы в список:

```python
# Первый и остальные
first, *rest = [1, 2, 3, 4, 5]
print(first)  # 1
print(rest)   # [2, 3, 4, 5]

# Первый, последний и всё между ними
first, *middle, last = [1, 2, 3, 4, 5]
print(first)   # 1
print(middle)  # [2, 3, 4]
print(last)    # 5

# Всё кроме последнего
*beginning, last = [10, 20, 30, 40]
print(beginning)  # [10, 20, 30]
print(last)       # 40
```

### `_` для ненужных значений

```python
# Нужны только имя и возраст
name, age, _ = ("Иван", 25, "Москва")

# Нужен только первый и последний
first, *_, last = [1, 2, 3, 4, 5]
print(first, last)  # 1 5

# Извлечь расширение файла
_, extension = "photo.jpg".rsplit(".", 1)
print(extension)  # jpg
```

## Практические примеры

### Обработка данных

```python
# Распаковка строки CSV
line = "Иван,25,ivan@example.com"
name, age, email = line.split(",")
print(f"Имя: {name}, Возраст: {age}, Email: {email}")

# Распаковка в цикле
points = [(1, 2), (3, 4), (5, 6)]
for x, y in points:
    print(f"x={x}, y={y}")

# Распаковка из словаря
user = {"name": "Иван", "age": 25}
for key, value in user.items():
    print(f"{key}: {value}")
```

### Функции, возвращающие несколько значений

```python
def get_min_max(numbers):
    """Возвращает минимум и максимум из списка"""
    return min(numbers), max(numbers)

minimum, maximum = get_min_max([5, 2, 8, 1, 9])
print(f"Минимум: {minimum}, Максимум: {maximum}")
```

## 🔎 Проверка на ходу

```python
# Что выведет этот код? Подумай, потом проверь в REPL
>>> a, b = 5, 10
>>> a, b = b, a + b
>>> print(a, b)
10 15  # a получил старое значение b (5→10), b получил сумму старых a+b (5+10=15)

>>> x, *y = "Hello"
>>> print(x, y)
'H' ['e', 'l', 'l', 'o']

# Один объект или два?
>>> a = [1, 2, 3]
>>> b = a
>>> a is b   # is проверяет: «это один и тот же объект?»
True

>>> c = [1, 2, 3]
>>> a is c   # Другой объект с таким же содержимым
False
>>> a == c   # Но значения равны
True
```

## ⚠️ Частые ошибки

```python
# ❌ Использование имени до связывания
# print(name)  # NameError: name 'name' is not defined
name = "Иван"
print(name)    # Теперь ок

# ❌ Путаница между = и ==
x = 5   # Связывание: привязать имя x к объекту 5
x == 5  # Сравнение: вернёт True

# ❌ Неожиданное разделение объекта
a = [1, 2, 3]
b = a             # Не копия! Тот же объект
b.append(4)       # a тоже «изменился»

# ✅ Если нужна копия
b = a.copy()      # Или b = a[:] или b = list(a)
```

## 💡 Главное

```python
# Имя — это бирка, не коробка
# = — это связывание, не «запись в ячейку»
# Тип принадлежит объекту, не имени
# Несколько имён могут указывать на один объект

a = [1, 2, 3]   # Создали объект [1, 2, 3], привязали имя a
b = a            # Привязали ещё одно имя b к тому же объекту
c = a.copy()     # Создали новый объект, привязали имя c

# a и b → один объект
# c → другой объект (с таким же содержимым)
```
