# 📝 Документирование функций
```toc
minLevel: 2
```

## Зачем документировать функции?

Код читается **гораздо чаще**, чем пишется. Хорошая документация помогает:
- Понять, что делает функция
- Узнать, какие параметры нужны
- Понять, что функция возвращает
- Избежать ошибок при использовании

**Правило:** Пишите код так, будто его будет читать злой психопат, который знает, где вы живёте!

## Docstring (строка документации)

**Docstring** — это строка документации, которая идёт сразу после определения функции. Python сохраняет её в специальном атрибуте `__doc__` и использует во встроенной функции `help()`.

### Базовый синтаксис

```python
def функция():
    """Это docstring — описание функции"""
    pass
```

**Важно:** 
- Используйте **тройные кавычки** `"""текст"""`
- Docstring должен быть **первой строкой** после `def`
- Для односторочных docstring используйте одну строку

## Односторочный docstring

Для простых функций:

```python
def square(x):
    """Возвращает квадрат числа"""
    return x * x
```

**Правила для однострочных docstring:**
- Начинается с заглавной буквы
- Заканчивается точкой
- Описывает действие в повелительном наклонении ("Возвращает...", "Вычисляет...", "Создаёт...")

## Многострочный docstring

Для более сложных функций:

```python
def calculate_discount(price, percent, minimum_price=0):
    """
    Вычисляет цену со скидкой.
    
    Применяет процент скидки к цене, но не позволяет
    цене опуститься ниже минимальной.
    
    Параметры:
        price (float): Исходная цена товара
        percent (float): Процент скидки (0-100)
        minimum_price (float): Минимально допустимая цена
    
    Возвращает:
        float: Цена со скидкой
    
    Примеры:
        >>> calculate_discount(1000, 20)
        800.0
        >>> calculate_discount(100, 50, minimum_price=60)
        60.0
    """
    discounted_price = price * (1 - percent / 100)
    return max(discounted_price, minimum_price)
```

**Структура многострочного docstring:**
1. Краткое описание (одна строка)
2. Пустая строка
3. Подробное описание
4. Параметры
5. Возвращаемое значение
6. Примеры использования

## Доступ к docstring

Python сохраняет docstring в специальном атрибуте `__doc__`:

```python
def greet(name):
    """Возвращает приветствие для указанного имени"""
    return f"Привет, {name}!"

print(greet.__doc__)
# Возвращает приветствие для указанного имени
```

Встроенная функция `help()` показывает docstring:

```python
help(greet)
# Help on function greet in module __main__:
# 
# greet(name)
#     Возвращает приветствие для указанного имени
```

## Doctest — тестирование через документацию

**Doctest** — это способ писать тесты прямо в docstring! Python может выполнить эти примеры и проверить результаты.

### Как работает doctest

```python
def sum_naturals(n):
    """
    Возвращает сумму натуральных чисел до n включительно.
    
    >>> sum_naturals(10)
    55
    >>> sum_naturals(100)
    5050
    """
    total, i = 0, 1
    while i <= n:
        total += i
        i += 1
    return total
```

**Синтаксис doctests:**
- `>>>` — начало команды (как в интерактивном режиме Python)
- Следующая строка (без `>>>`) — ожидаемый результат

### Запуск doctests

```python
if __name__ == '__main__':
    from doctest import testmod
    testmod()
```

Если тесты проходят — ничего не выводится. Если есть ошибки — показывается подробный отчёт.

Также можно запустить из командной строки:

```bash
# Запуск doctests для конкретного файла
python -m doctest my_module.py

# С подробным выводом
python -m doctest -v my_module.py
```

## 🎯 Практические примеры

### Пример 1: Простая функция

```python
def is_even(number):
    """
    Проверяет, является ли число чётным.
    
    Args:
        number (int): Проверяемое число
    
    Returns:
        bool: True, если число чётное, иначе False
    
    Examples:
        >>> is_even(4)
        True
        >>> is_even(7)
        False
    """
    return number % 2 == 0
```

### Пример 2: Функция с несколькими параметрами

```python
def find_in_range(numbers, minimum, maximum):
    """
    Находит элементы списка в заданном диапазоне.
    
    Args:
        numbers (list): Список чисел для поиска
        minimum (float): Минимальное значение диапазона (включительно)
        maximum (float): Максимальное значение диапазона (включительно)
    
    Returns:
        list: Новый список с элементами из диапазона
    
    Examples:
        >>> find_in_range([1, 5, 10, 15, 20], 5, 15)
        [5, 10, 15]
        >>> find_in_range([1, 2, 3], 10, 20)
        []
    """
    return [x for x in numbers if minimum <= x <= maximum]
```

### Пример 3: Функция с обработкой ошибок

```python
def safe_divide(dividend, divisor):
    """
    Выполняет деление с обработкой деления на ноль.
    
    Args:
        dividend (float): Число, которое делим
        divisor (float): Число, на которое делим
    
    Returns:
        float or None: Результат деления или None при делении на ноль
    
    Examples:
        >>> safe_divide(10, 2)
        5.0
        >>> safe_divide(10, 0) is None
        True
    """
    if divisor == 0:
        return None
    return dividend / divisor
```

### Пример 4: Функция с изменяемыми значениями по умолчанию

```python
def create_profile(name, tags=None):
    """
    Создаёт профиль пользователя.
    
    Args:
        name (str): Имя пользователя
        tags (list, optional): Список тегов. По умолчанию None (создаётся пустой список)
    
    Returns:
        dict: Словарь с данными профиля
    
    Examples:
        >>> profile = create_profile("Anna")
        >>> profile['name']
        'Anna'
        >>> profile['tags']
        []
        >>> profile2 = create_profile("Ivan", ["programmer", "python"])
        >>> profile2['tags']
        ['programmer', 'python']
    """
    if tags is None:
        tags = []
    
    return {
        "name": name,
        "tags": tags
    }
```

## Продвинутые примеры doctests

### Тестирование исключений

```python
def divide(a, b):
    """
    Делит a на b.
    
    >>> divide(10, 2)
    5.0
    >>> divide(10, 0)
    Traceback (most recent call last):
        ...
    ZeroDivisionError: division by zero
    """
    return a / b
```

### Тестирование с многострочным выводом

```python
def print_list(items):
    """
    Выводит элементы списка построчно.
    
    >>> print_list([1, 2, 3])
    1
    2
    3
    """
    for item in items:
        print(item)
```

### Тестирование с приблизительным сравнением

```python
def average(numbers):
    """
    Вычисляет среднее значение.
    
    >>> average([1, 2, 3])
    2.0
    >>> round(average([1, 2, 3, 4]), 2)
    2.5
    """
    return sum(numbers) / len(numbers)
```

## Стили документирования

### 1. Google Style (рекомендуется)

```python
def function(param1, param2):
    """
    Краткое описание функции.
    
    Подробное описание функции,
    может занимать несколько строк.
    
    Args:
        param1 (int): Описание первого параметра
        param2 (str): Описание второго параметра
    
    Returns:
        bool: Описание возвращаемого значения
    
    Raises:
        ValueError: Когда возникает эта ошибка
    
    Examples:
        >>> function(1, "test")
        True
    """
    pass
```

### 2. NumPy/SciPy Style

```python
def function(param1, param2):
    """
    Краткое описание функции.
    
    Подробное описание функции.
    
    Parameters
    ----------
    param1 : int
        Описание первого параметра
    param2 : str
        Описание второго параметра
    
    Returns
    -------
    bool
        Описание возвращаемого значения
    """
    pass
```

### 3. reStructuredText (Sphinx)

```python
def function(param1, param2):
    """
    Краткое описание функции.
    
    :param param1: Описание первого параметра
    :type param1: int
    :param param2: Описание второго параметра
    :type param2: str
    :returns: Описание возвращаемого значения
    :rtype: bool
    """
    pass
```

## 💡 Лучшие практики

### ✅ Хорошие практики:

1. **Всегда пишите docstring для публичных функций**
2. **Начинайте с краткого описания** — что делает функция
3. **Документируйте все параметры** — типы и назначение
4. **Указывайте возвращаемое значение** — тип и смысл
5. **Добавляйте примеры** — особенно для сложных функций
6. **Обновляйте документацию** при изменении кода
7. **Документируйте исключения** — когда и какие могут возникнуть

### ❌ Плохие практики:

1. Документация, которая просто повторяет код
   ```python
   def add(a, b):
       """Складывает a и b"""  # Это очевидно!
       return a + b
   ```

2. Устаревшая документация
3. Слишком длинные или слишком короткие описания
4. Отсутствие примеров для сложных функций

## 🔎 Проверка на ходу

Используй встроенные инструменты Python:

```python
>>> def add(a, b):
...     """Возвращает сумму двух чисел"""
...     return a + b
...
>>> add.__doc__
'Возвращает сумму двух чисел'
>>> help(add)
Help on function add in module __main__:

add(a, b)
    Возвращает сумму двух чисел
```

## Резюме

- **Docstring** — строка документации после `def`
- Используйте **тройные кавычки** `""" """`
- **Однострочные** для простых функций
- **Многострочные** для сложных (с параметрами, примерами)
- **Doctest** — тесты прямо в документации (синтаксис `>>>`)
- Запуск: `from doctest import testmod; testmod()` или `python -m doctest file.py`
- Документируйте: что делает, параметры, возвращаемое значение, примеры
- Обновляйте документацию вместе с кодом
- Используйте единый стиль (Google Style рекомендуется для начинающих)
