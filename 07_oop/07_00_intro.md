# 🏛️ Основы ООП

Добро пожаловать в раздел об объектно-ориентированном программировании! Здесь вы найдёте полное руководство по ООП в Python — от базовых концепций до практических паттернов.

## 🧠 Четыре принципа ООП

ООП держится на четырёх фундаментальных принципах. Они не придуманы ради красоты — каждый решает конкретную проблему разработки.

### Абстракция

Выделяем только важные характеристики объекта, скрывая несущественные детали.

Когда вы вызываете `list.append()` — вас не интересует как именно Python перераспределяет память. Вы работаете с абстракцией «список», а не с деталями реализации.

```python
class Car:
    def start(self):
        return "Машина едет"
    # Не важно: как работает двигатель, какое давление в цилиндрах, etc.
```

### Инкапсуляция

Объединяем данные и методы, работающие с этими данными, в одну сущность — класс. Объект сам управляет своим состоянием.

```python
# Без инкапсуляции — данные и логика разбросаны
balance = 1000
def deposit(amount): ...

# С инкапсуляцией — всё вместе
class BankAccount:
    def __init__(self, balance):
        self.balance = balance
    def deposit(self, amount):
        self.balance += amount
```

Инкапсуляция ≠ сокрытие данных. Сокрытие (`_`, `__`, `@property`) — это дополнительный инструмент поверх инкапсуляции. Подробнее: [[07_04_encapsulation|Инкапсуляция]].

### Наследование

Новый класс строится на основе существующего, получая его атрибуты и методы. Позволяет переиспользовать код и строить иерархии.

```python
class Animal:
    def eat(self): ...

class Dog(Animal):      # Dog получает eat() бесплатно
    def bark(self): ...
```

Подробнее: [[07_05_inheritance|Наследование]], [[07_11_mro|MRO]].

### Полиморфизм

Разные объекты отвечают на один и тот же вызов по-своему. Один интерфейс — разное поведение.

```python
for shape in [Circle(5), Rectangle(4, 6)]:
    print(shape.area())   # Каждый считает площадь по-своему
```

Подробнее: [[07_06_polymorphism|Полиморфизм]], [[07_10_abc_and_protocol|ABC и Protocol]].

## 📝 Шпаргалка

### Базовый синтаксис класса

```python
class ИмяКласса:
    """Документация класса"""

    class_attribute = "общее для всех"  # Атрибут класса

    def __init__(self, param):
        self.param = param              # Атрибут экземпляра

    def method(self):
        return self.param

    @classmethod
    def from_something(cls, data):      # Альтернативный конструктор
        return cls(data)

    @staticmethod
    def validate(value):                # Вспомогательная функция
        return value is not None
```

### Наследование

```python
class Child(Parent):
    def __init__(self, param, extra):
        super().__init__(param)
        self.extra = extra
```

### Абстрактный класс

```python
from abc import ABC, abstractmethod

class Base(ABC):
    @abstractmethod
    def do_something(self): ...
```

### Protocol

```python
from typing import Protocol

class Drawable(Protocol):
    def draw(self) -> None: ...
```

### dataclass

```python
from dataclasses import dataclass, field

@dataclass
class Point:
    x: float
    y: float
    tags: list = field(default_factory=list)
```

### `__slots__`

```python
class Point:
    __slots__ = ("x", "y")

    def __init__(self, x, y):
        self.x = x
        self.y = y
```
