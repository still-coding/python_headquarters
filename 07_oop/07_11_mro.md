## Что такое MRO?

**MRO** (Method Resolution Order) — порядок, в котором Python ищет метод при его вызове. Для обычного наследования это очевидно: сначала в самом классе, потом в родителе. Но при множественном наследовании возникает вопрос: в каком порядке обходить родителей?

```python
class A:
    def hello(self):
        return "A"

class B(A):
    def hello(self):
        return "B"

class C(A):
    def hello(self):
        return "C"

class D(B, C):   # Наследует и B, и C — у обоих есть hello()
    pass

d = D()
print(d.hello())  # Чей hello() вызовется?
```

Python решает это через MRO.

## Посмотреть MRO

Каждый класс хранит свой MRO — его можно посмотреть:

```python
print(D.mro())
# [<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>]

# Или через атрибут
print(D.__mro__)
```

Python ищет метод слева направо по этому списку — и останавливается на первом совпадении. Поэтому `d.hello()` вернёт `"B"` — `B` стоит раньше `C`.

## Проблема ромба

Классическая проблема множественного наследования — **diamond problem** (проблема ромба):

```
    A
   / \
  B   C
   \ /
    D
```

```python
class A:
    def hello(self):
        return "A"

class B(A):
    def hello(self):
        return "B"

class C(A):
    def hello(self):
        return "C"

class D(B, C):
    pass

print(D.mro())
# [D, B, C, A, object]

print(D().hello())  # "B" — нашли в B, дальше не идём
```

Без MRO было бы непонятно: брать `hello()` из `B` или из `C`? Python решает это однозначно через алгоритм C3.

## Алгоритм C3

Python использует алгоритм линеаризации **C3** — он гарантирует:
- Дочерний класс всегда стоит **раньше** родительских
- Порядок родителей из объявления **сохраняется**
- Монотонность — если `B` стоит раньше `C` в одном классе, так будет и во всех потомках

```python
class D(B, C):   # B указан раньше C
    pass

# MRO: [D, B, C, A, object]
#           ↑  ↑
#           B раньше C — как и в объявлении
```

### Когда C3 не может построить MRO

Иногда Python не может построить корректный MRO и выбрасывает ошибку:

```python
class A:
    pass

class B(A):
    pass

class C(A, B):   # Здесь A стоит до B, но B наследует A
    pass          # Это противоречие — A должен быть после B

# TypeError: Cannot create a consistent method resolution order
```

Это защита от противоречивых иерархий.

## `super()` и MRO

`super()` работает именно по MRO — он вызывает **следующий класс в цепочке**, а не обязательно прямого родителя:

```python
class A:
    def hello(self):
        print("A")

class B(A):
    def hello(self):
        print("B")
        super().hello()   # Следующий по MRO — A

class C(A):
    def hello(self):
        print("C")
        super().hello()   # Следующий по MRO — A

class D(B, C):
    def hello(self):
        print("D")
        super().hello()   # Следующий по MRO — B


d = D()
d.hello()
# D
# B
# C    ← не A! super() в B вызывает C, потому что MRO: [D, B, C, A]
# A
```

Это называется **cooperative multiple inheritance** — все классы в цепочке вызывают `super()` и передают управление дальше по MRO. Поэтому важно всегда использовать `super()` вместо прямого вызова родителя.

## Практический пример

```python
class Flyable:
    def move(self):
        print("Лечу")
        super().move()   # Передаём дальше по MRO

class Swimmable:
    def move(self):
        print("Плыву")
        super().move()   # Передаём дальше по MRO

class Base:
    def move(self):
        print("Двигаюсь")  # Базовый случай — цепочка заканчивается здесь

class Duck(Flyable, Swimmable, Base):
    def move(self):
        print("Утка:")
        super().move()


print(Duck.mro())
# [Duck, Flyable, Swimmable, Base, object]

Duck().move()
# Утка:
# Лечу
# Плыву
# Двигаюсь
```

## 🔎 Проверка на ходу

```python
>>> class A:
...     pass
>>> class B(A):
...     pass
>>> class C(A):
...     pass
>>> class D(B, C):
...     pass
>>>
>>> D.mro()
[<class 'D'>, <class 'B'>, <class 'C'>, <class 'A'>, <class 'object'>]
>>>
>>> # Все классы наследуют object
>>> int.mro()
[<class 'int'>, <class 'object'>]
>>> bool.mro()
[<class 'bool'>, <class 'int'>, <class 'object'>]
```

## Резюме

- **MRO** — порядок, в котором Python ищет методы при наследовании
- Посмотреть: `ClassName.mro()` или `ClassName.__mro__`
- Python использует алгоритм **C3**: дочерний класс раньше родителей, порядок родителей из объявления сохраняется
- **Проблема ромба** решается однозначно — через MRO
- `super()` вызывает следующий класс по MRO, а не прямого родителя — поэтому `super()` нужно использовать везде в цепочке
- Если построить корректный MRO невозможно — Python выбросит `TypeError`
