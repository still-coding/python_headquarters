## Зачем нужны модули и пакеты?

Когда программа маленькая — она помещается в один файл. Но как только появляются десятки функций и классов, один файл превращается в свалку, в которой невозможно что-либо найти.

**Модули** позволяют разбить код на отдельные файлы по смыслу: отдельно — работа с базой данных, отдельно — математика, отдельно — взаимодействие с пользователем. **Пакеты** — это папки из модулей, которые объединяют связанные модули в одну логическую единицу.

```python
# ❌ Всё в одном файле main.py — каша на 2000 строк
def connect_to_db(): ...
def calculate_taxes(): ...
def send_email(): ...
def parse_csv(): ...
# ...ещё 50 функций

# ✅ Разбито по модулям
# database.py    — работа с БД
# taxes.py       — налоговые расчёты
# notifications.py — уведомления
# importers.py   — парсинг данных
```

Кроме собственного кода, модули позволяют использовать **чужой**: стандартная библиотека Python и тысячи сторонних пакетов с [PyPI](https://pypi.org) подключаются тем же самым `import`.

## Три уровня работы с чужим кодом

```python
# 1. Стандартная библиотека — идёт вместе с Python
import math
import json
from datetime import datetime

# 2. Сторонние пакеты — устанавливаются через pip или uv
import requests        # HTTP-запросы
import numpy as np     # Научные вычисления

# 3. Свой код — твои собственные модули и пакеты
from myapp.database import connect
from myapp.utils import format_date
```

## 💡 Ключевые идеи раздела

- **Модуль** — это `.py` файл
- **Пакет** — это папка с модулями (исторически с файлом `__init__.py`)
- **`import`** не просто копирует код — он запускает модуль и сохраняет результат в кэше
- **`sys.path`** определяет, где Python ищет модули — не магия, а обычный список папок
- **Виртуальное окружение** изолирует зависимости одного проекта от других
