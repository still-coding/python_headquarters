# 🚨 Исключения

Добро пожаловать в раздел об обработке исключений! Здесь вы узнаете как Python сигнализирует об ошибках, как их перехватывать, и как писать надёжный код который корректно работает даже в нештатных ситуациях.

## 🧠 Зачем нужна обработка исключений?

Программы работают в реальном мире — а реальный мир непредсказуем:
- Файл которого нет
- Сеть которая пропала
- Пользователь который ввёл текст вместо числа
- Деление на ноль

Без обработки исключений программа просто падает. С обработкой — она может среагировать, сообщить об ошибке и продолжить работу.

```python
# Без обработки — программа падает
number = int(input("Введите число: "))   # Пользователь вводит "abc" → crash

# С обработкой — программа выживает
try:
    number = int(input("Введите число: "))
except ValueError:
    print("Это не число, попробуйте ещё раз")
```

## 📝 Шпаргалка

### Базовый синтаксис

```python
try:
    # Код который может упасть
    result = int(input("Число: "))
except ValueError:
    # Реакция на конкретное исключение
    print("Не число!")
except (TypeError, AttributeError):
    # Несколько исключений сразу
    print("Неверный тип")
except Exception as e:
    # Любое исключение + доступ к объекту
    print(f"Ошибка: {e}")
else:
    # Выполняется если исключения НЕ было
    print(f"Результат: {result}")
finally:
    # Выполняется всегда
    print("Готово")
```

### Вызов исключения

```python
raise ValueError("Значение должно быть положительным")
```

### Собственное исключение

```python
class AppError(Exception):
    pass

class ValidationError(AppError):
    def __init__(self, field, message):
        self.field = field
        super().__init__(f"{field}: {message}")
```
