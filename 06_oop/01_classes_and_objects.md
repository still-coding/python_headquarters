# 🧱 Классы и объекты
```toc
minLevel: 2
```

## Что такое объект?

В реальном мире всё вокруг — это объекты: стул, кошка, банковский счёт, автомобиль. У каждого объекта есть:
- **Характеристики** (данные) — цвет кошки, баланс счёта, марка машины
- **Поведение** (действия) — кошка мяукает, счёт пополняется, машина едет

В программировании **объект** — это такая же сущность: данные + действия над ними, упакованные вместе.

## Что такое класс?

**Класс** — это шаблон (чертёж), по которому создаются объекты.

Аналогия: класс — это рецепт торта, объект — конкретный испечённый торт. По одному рецепту можно испечь сколько угодно тортов, и каждый будет отдельным объектом.

```
Класс Cat          Объекты (экземпляры)
─────────────      ────────────────────
имя: ?         →   имя: "Мурка"
цвет: ?        →   имя: "Барсик"
мяукать()      →   имя: "Персик"
```

## Синтаксис создания класса

```python
class Cat:
    """Класс, описывающий кошку"""
    pass  # Пустое тело — класс пока ничего не делает
```

**Ключевые моменты:**
- `class` — ключевое слово для создания класса
- `Cat` — имя класса, по соглашению пишется с **заглавной буквы** (CamelCase)
- Тело класса — с отступом, как у функции

## Создание объекта (экземпляра)

```python
class Cat:
    pass

# Создаём объекты — вызываем класс как функцию
cat1 = Cat()
cat2 = Cat()

print(type(cat1))  # <class '__main__.Cat'>
print(cat1)        # <__main__.Cat object at 0x...>

# Это два разных объекта
print(cat1 is cat2)  # False
```

## Атрибуты объекта

Атрибуты — это данные, хранящиеся внутри объекта. Их можно добавить прямо после создания:

```python
class Cat:
    pass

cat = Cat()
cat.name = "Мурка"
cat.color = "рыжая"
cat.age = 3

print(cat.name)   # Мурка
print(cat.color)  # рыжая
print(cat.age)    # 3
```

Но это неудобно — каждый раз нужно вручную задавать атрибуты. Правильный способ — через `__init__`, который мы разберём в файле [[03_init_and_self|`__init__` и `self`]].

## Полный пример: класс с `__init__`

Вот как выглядит настоящий класс — не беспокойтесь пока о деталях `__init__` и `self`, они подробно разобраны дальше:

```python
class Cat:
    """Класс, описывающий кошку"""

    def __init__(self, name, color, age):
        self.name = name
        self.color = color
        self.age = age

    def meow(self):
        return f"{self.name} говорит: Мяу!"

    def describe(self):
        return f"{self.name} — {self.color} кошка, {self.age} лет"


# Создаём объекты
cat1 = Cat("Мурка", "рыжая", 3)
cat2 = Cat("Барсик", "серый", 5)

print(cat1.name)        # Мурка
print(cat2.name)        # Барсик
print(cat1.meow())      # Мурка говорит: Мяу!
print(cat2.describe())  # Барсик — серый кошка, 5 лет

# Каждый объект хранит свои данные независимо
print(cat1.age)  # 3
print(cat2.age)  # 5
```

## Класс — это тоже объект

В Python всё является объектом, включая сам класс:

```python
class Cat:
    pass

print(type(Cat))   # <class 'type'>
print(Cat)         # <class '__main__.Cat'>

# Класс можно присвоить переменной
Animal = Cat
animal = Animal()
print(type(animal))  # <class '__main__.Cat'>
```

## 🎯 Пример с реальной задачей

Смоделируем банковский счёт:

```python
class BankAccount:
    """Простой банковский счёт"""

    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        """Пополнить счёт"""
        self.balance += amount
        return self.balance

    def withdraw(self, amount):
        """Снять деньги"""
        if amount > self.balance:
            return "Недостаточно средств"
        self.balance -= amount
        return self.balance

    def get_info(self):
        return f"Счёт {self.owner}: {self.balance} руб."


# Создаём счета
account1 = BankAccount("Анна", 1000)
account2 = BankAccount("Иван")

account1.deposit(500)
account1.withdraw(200)
print(account1.get_info())  # Счёт Анна: 1300 руб.

account2.deposit(300)
print(account2.get_info())  # Счёт Иван: 300 руб.

# Счета независимы друг от друга
print(account1.balance)  # 1300
print(account2.balance)  # 300
```

## Зачем нужны классы?

| Без классов | С классами |
|---|---|
| Данные разбросаны по переменным | Данные сгруппированы внутри объекта |
| Функции никак не связаны с данными | Методы — часть объекта |
| Сложно создать несколько «одинаковых» сущностей | Легко создать много объектов по одному шаблону |
| Код трудно читать и поддерживать | Код организован и понятен |

## 🔎 Проверка на ходу

```python
>>> class Dog:
...     pass
...
>>> dog = Dog()
>>> dog.name = "Рекс"
>>> dog.name
'Рекс'
>>> type(dog)
<class '__main__.Dog'>
>>> isinstance(dog, Dog)
True
```

## Резюме

- **Класс** — шаблон для создания объектов
- **Объект** — конкретный экземпляр класса
- Имена классов пишутся в стиле `CamelCase`
- Создание объекта: `obj = ClassName()`
- Каждый объект хранит свои данные независимо от других
- Объекты объединяют данные и поведение вместе
